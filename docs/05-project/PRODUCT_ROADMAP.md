## 设计规范

### Apple Human Interface Guidelines

**核心原则**：
1. **Deference** - 内容优先
2. **Clarity** - 清晰易读
3. **Depth** - 层次分明

**实施要点**：
- 使用 SF Symbols（或蒙古文图标）
- 遵循 iOS 导航模式
- 支持动态字体大小
- 支持深色模式
- 支持 VoiceOver（无障碍）

**蒙古文特殊考虑**：
- 竖排文字适配
- 从右到左布局（部分场景）
- 特殊字符渲染

### Material Design 3 (Android)

**核心原则**：
1. **Personalized** - 个性化
2. **Adaptive** - 自适应
3. **Accessible** - 无障碍

**实施要点**：
- 使用 Material 3 组件
- 遵循 Android 导航
- 支持动态颜色
- 支持多窗口
- 支持 TalkBack

### Web/PWA 标准

**核心要求**：
- 响应式设计
- 离线支持（Service Worker）
- 可安装（Web App Manifest）
- HTTPS 加密
- 快速加载（< 3s）

**性能指标**：
- First Contentful Paint < 1.5s
- Time to Interactive < 3.5s
- Cumulative Layout Shift < 0.1

## 发布流程

### 版本控制

**Git 分支策略**：
```
main (生产)
  ↑
develop (开发)
  ↑
feature/* (功能)
  ↑
bugfix/* (修复)
```

**发布流程**：
1. 功能开发（feature 分支）
2. 代码审查（PR）
3. 测试验证（CI/CD）
4. 合并到 develop
5. 发布候选（release 分支）
6. 最终测试
7. 合并到 main
8. 打标签（tag）
9. 发布

### App Store 发布

**iOS App Store**：
- 遵循 App Store 审核指南
- 准备截图和预览视频
- 编写应用描述
- 设置价格和地区
- 提交审核（1-3 天）

**Google Play**：
- 遵循开发者计划政策
- 准备素材
- 内容分级
- 发布（几小时）

**Web 发布**：
- 域名和 SSL
- CDN 部署
- 监控和日志
- 持续部署
